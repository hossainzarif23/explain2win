Explain2Win system architecture

A voice-first learning system that turns a spoken explanation into rubric-based feedback, targeted retrieval practice, repeated-attempt evidence, and a personal knowledge graph.

Primary user Independent learner proving understanding by teaching aloud.
Core value Convert passive familiarity into observable mastery evidence.
Mastery rule Two consecutive 9/10 explanations with perfect quizzes.
Binding constraint External AI, storage, database, and browser microphone availability.

System context and trust boundary

Interactive state stays in the browser. Sensitive credentials, persistence, AI calls, uploads, and billing operations remain inside the authenticated Next.js server boundary.

Client / identity Product UI Application server Persistent storage AI service

Learning and mastery pipeline

The critical runtime path combines browser media, two dedicated HTTP handlers, tRPC domain mutations, Gemini workloads, and a durable two-attempt mastery state machine.

The dashed path is the deliberate practice loop. Automatic graph expansion is asynchronous and does not block quiz completion.

Component architecture

The repository is a modular monolith: one deployable Next.js application, separated into presentation, transport, domain, adapter, and persistence responsibilities.

Critical state boundaries

  • StudySession owns the fixed topic, scope, and mastery streak.
  • Explanation is one evaluated voice attempt.
  • Each explanation can have exactly one QuizSession.
  • Attempt comparisons are cached by earlier/later pair.
  • S3 keys are durable; playback URLs expire after one hour.

Current risk: some multi-step writes and ownership checks are not yet production-hardened. See the quality drivers below.

Deployment view

The application requires a Node-capable Next.js host and outbound access to four managed services. It is not an edge-only deployment because Prisma, SDKs, raw uploads, and server credentials run in the Node runtime.

Recommended host class: stateful Node/serverless Node, not an edge-only runtime. Database, Gemini, and S3 are mandatory for the complete learning loop; Stripe is gated.
